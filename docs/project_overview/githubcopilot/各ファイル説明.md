# 現状作成済みの、```.github/```フォルダ配下のGithub Copilot設定ファイルの使い方の説明

### Agents
- ```.github/agents/folder_specific_agent.md```
    - コンテキストを押さえて、特定のフォルダに特化させたエージェントです
    - Agentをドロップダウンから選んだ上で、「target_directory=目的のフォルダパス」をプロンプト入力欄に指定してください。
    - 目的のフォルダ配下に、```.github/copilot-instructions.md```が存在する場合に特に威力を発揮するため、事前に目的のフォルダ配下に```.github/copilot-instructions.md```を作成しておくことを推奨します。
        - ```.github/prompts/1_create_instructions.prompt.md```を利用すると良いです。

### Instructions
- ```.github/instructions/modern.instructions.md```
    - モダンなPythonコードベースに関する一般的な指示をまとめたファイルです。
    - Pythonコードベースの理解を助けるために、pyファイルを対象として自動的に読み込まれます。
- ```.github/instructions/react.instructions.md```
    - 現状REACTをCDNで読み込む形で利用している。それに沿った形でコードを生成・編集するための指示をまとめたファイルです。
    - js/jsxファイルを対象として自動的に読み込まれます

### Prompts
- ```.github/prompts/1_create_instructions.prompt.md```
    - 特定のフォルダ専用の```.github/copilot-instructions.md```をAIに作成させるためのプロンプトテンプレートです。
    - copilot専用の内容を記述するために、英語で作成させています。
    - スラッシュコマンドでこのプロンプトを選んだ上で、「target_directory=目的のフォルダパス」をプロンプト入力欄に指定してください。
    - 複数ファイルを確認することによる、コンテキスト肥大を防ぐために、サブエージェントを利用してコードベース全体を調査するように促しています。
- ```.github/prompts/2_create_readme.prompt.md```
    - 特定のフォルダ専用のREADME.mdをAIに作成させるためのプロンプトテンプレートです。
    - スラッシュコマンドでこのプロンプトを選んだ上で、「target_directory=目的のフォルダパス」をプロンプト入力欄に指定してください。
    - 人間が読むことを想定して、日本語で詳細にはあまり立ち入らない形で、簡潔にまとめるように促しています。
    - ```.github/copilot-instructions.md```があることを想定しているため、このプロンプト実行前に、```.github/prompts/1_create_instructions.prompt.md```を利用して、目的のフォルダ配下に```.github/copilot-instructions.md```を作成しておくことを推奨します。
- ```.github/prompts/3_review.prompt.md```
    - 特定のフォルダ専用のコードレビューをAIに実行させるためのプロンプトテンプレートです。
    - スラッシュコマンドでこのプロンプトを選んだ上で、「target_directory=目的のフォルダパス」をプロンプト入力欄に指定してください。
    - 目的のフォルダ配下に、```.github/copilot-instructions.md```が存在する場合に特に威力を発揮するため、事前に目的のフォルダ配下に```.github/copilot-instructions.md```を作成しておくことを推奨します。
        - ```.github/prompts/1_create_instructions.prompt.md```を利用すると良いです。
    - レビューのガイドラインとして、```.github/review_guideline.md```を目的のフォルダ配下に作成しておくことを推奨します。
        - ガイドラインの例として、```.github/review_guideline.md.example```を参照してください。
    - レビューは、```.github/review/```フォルダ配下に作成されます。

# このプロジェクトにおけるGithub Copilotの使い方

## 基本的な考え方
- ```.github/agents/```
    - 特定のツールや、汎用的なカスタム指示（特定のファイル群の参照・特定の操作実行指示など）を持ったエージェントを定義します。
    - promptsファイルは、1つしか一度に使えないため（もちろんコピペすれば複数使えますが）、プロンプトで、個別の指示を与え、ある程度汎用的な指示は、エージェント側で持たせる形が良いです。
- ```.github/instructions/```・```.github/copilot-instructions.md```
    - これらは、自動的に読み込まれるファイル群です
    - ```.github/copilot-instructions.md```は常に読み込まれるので、プロジェクト全体像に関する概要をコンテキストを極力抑えながら記載しておくと良いです。
    - ```.github/instructions/```以下のファイルは、GLOBパターンを有効に活用して、特定のファイル群に関する指示を分割して記載しておくと良いです。
        - pythonファイルに関する指示、hoge/フォルダ配下のファイルに関する指示、など。
- ```.github/prompts/```
    - よく使う、個別のプロンプトを保存しておく場所です。
    - 例えば、コードレビュー用のプロンプト、テストコード生成用のプロンプト、ドキュメント生成用のプロンプトなどを保存しておくと良いです。

## このプロジェクト固有の使い方

### ルート直下のGithub Copilot設定
- ```.github/agents/```
    - なるべく、${input:key:value}を使って、汎用的なエージェントを作成してください。
    - 何度も似たような値を代入する必要がある場合は、${input:key:value}を置換したバージョンのエージェントを作成して構いません。
        - ただし、その場合は、ファイル名の先頭にtmp_をつけて、一時的なエージェントであることを明示してください。　（tmpを先頭につけるファイルは、GIT管理から外します）
        - その際に、エージェント名を異なる名前に変更するのを忘れないでください。（名前は任意）
- ```.github/prompts/```
    - なるべく、${input:key:value}を使って、汎用的なプロンプトを作成してください
    - agentの場合と同様に、何度も似たような値を代入する必要がある場合は、${input:key:value}を置換したバージョンのプロンプトを作成して構いません。
        - ただし、その場合は、ファイル名の先頭にtmp_をつけて、一時的なプロンプトであることを明示してください。　（tmpを先頭につけるファイルは、GIT管理から外します）
        - その際に、プロンプト名を異なる名前に変更するのを忘れないでください。（名前は任意）
- ```.github/instructions/```・```.github/copilot-instructions.md```
    - 基本的に、極力勝手に変更を加えないようにしてください。
    - これらのINSTRUCTIONファイルは、全員共通のものを利用する形が望ましいです。
    - ```.github/copilot-instructions.md```については、最新の情報と極力合わせたいです。
        - しかし、細かい点は記載するべきでなく、あくまでプロジェクト全体に関わる概要に留めてください。
        - ある程度は、作業フォルダ・作業サービス特有のエージェントの作成で対応するのがベストです。
        - このファイルの更新には、専用のワークフローを用意する予定です。

### 各作業フォルダ直下のGithub Copilot設定
- 各作業フォルダ直下に```.github/```フォルダを作成して、Github Copilot関連の設定・出力をまとめてください。

# GitHub Copilot プロンプト機能

プロンプトは、特定のタスクに特化したテンプレートを定義する機能です。

---

## 📖 概要

### プロンプトとは

プロンプトは、よく使うタスクをテンプレート化したものです：

- **再利用可能**: 同じ指示を何度も書く必要がない
- **標準化**: チーム内で同じプロンプトを共有
- **カスタマイズ**: プロジェクト固有の要件を組み込み可能

### 用途

- コードリファクタリングの依頼
- テストコード生成の依頼
- ドキュメント生成の依頼
- 特定フォーマットでの出力依頼

---

## 📁 ファイル仕様

### 配置場所

```
.github/prompts/
└── {prompt-name}.prompt.md
```

> **注意**: `.github/prompts/` 直下に配置する必要があります。サブフォルダは使用できません。

### ファイル名規則

```
{prompt-name}.prompt.md
```

**例**:
- `refactor-code.prompt.md`
- `generate-tests.prompt.md`
- `review-security.prompt.md`

### 呼び出し方

Copilot Chatでスラッシュコマンドを使用：

```
/prompt-name
```

**例**: `/refactor-code`, `/generate-tests`

---

## 📝 ファイル構成

### 基本構造

```markdown
---
agent: agent-name
---

プロンプトの内容（指示文）
```

### フロントマター

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `agent` | ❌ | 使用するエージェント名 |

### フロントマター例

```yaml
---
agent: code-reviewer.default
---
```

### プロンプト本文

Markdown形式で指示内容を記述します。

```markdown
---
agent: code-reviewer.default
---

このファイルのリファクタリングを実施してください。

以下の観点でチェック：
- PEP 8準拠
- 型ヒントの追加
- docstringの充実化
- 関数の適切な分割

改善点があれば、具体的なコード修正案を提示してください。
```

## 💡 活用例

### コードレビュー用プロンプト

```markdown
---
agent: code-reviewer.default
---

選択したコードをレビューしてください。

**レビュー観点:**
- セキュリティリスク
- パフォーマンス問題
- コーディング規約違反
- エラーハンドリング

**出力形式:**
1. 問題点の一覧（重要度順）
2. 各問題の改善案
3. 全体的な評価
```

### テスト生成用プロンプト

```markdown
---
agent: test-generator.default
---

選択した関数/クラスのユニットテストを生成してください。

**テスト要件:**
- pytestを使用
- 正常系・異常系の両方をカバー
- エッジケースを含む
- モックが必要な場合は適切に使用
```

### ドキュメント生成用プロンプト

```markdown
---
agent: doc-generator.default
---

選択したコードのドキュメントを生成してください。

**出力内容:**
- 関数/クラスの概要説明
- 引数・戻り値の説明
- 使用例
- 注意事項
```

## 💡 ベストプラクティス

### 1. 具体的な指示を記述

曖昧な指示ではなく、具体的な要件を記述します。

**良い例**: 「PEP 8準拠をチェックし、違反箇所を指摘」
**悪い例**: 「コードを良くして」

### 2. 出力形式を指定

期待する出力形式を明示することで、一貫した結果が得られます。

### 3. エージェントと組み合わせ

適切なエージェントを指定することで、より専門的な応答が得られます。

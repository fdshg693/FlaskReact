## 🎯 タスクの追加・更新ルール

### 新しいタスクを追加する時

1. **タスクID**: `カテゴリ-番号` 形式で命名
   - INFRA: インフラ・構造
   - REFACTOR: リファクタリング
   - ARCH: アーキテクチャ
   - FEAT: 機能追加
   - DX: 開発体験
   - TEST: テスト・品質
   - DOC: ドキュメント

2. **必須項目**:
   - 状態: 未着手 / 進行中 / レビュー中 / 完了
   - 説明: 何をするのか簡潔に
   - 理由: なぜ必要か

3. **推奨項目**:
   - 影響範囲: どこが変わるか
   - 推定工数: どれくらいかかるか
   - 関連課題: problems/ の該当ファイル

### タスクを完了した時

1. 完了日とともに「最近完了したタスク」セクションに移動
2. 1ヶ月以上前の完了タスクは削除（必要なら problems/ に記録）

### バックログとの使い分け

- **TODOS（このファイル）**: 近いうちに着手する具体的なタスク
- **problems/**: 体系的な課題管理、優先度付け、長期的な計画
- **バックログ**: いつか実現したいアイデア、現時点では優先度が低い

# 開発効率を上げるために、作成することが望ましいカスタムスクリプト


- import系のエラーが出ないように、全IMPORT文をチェックするスクリプト（MYPY・RUFF・PYLANCEなどで出来るならそれでもいい）
    - 基本的にSRCから順番に遡るだけなので、かなり簡単に作成できるはず
- ドキュメント劣化チェック
    - ドキュメントが取り扱っている一連のファイルをドキュメント下部などに記載して、スクリプトの実行時にそれらのファイルが存在するかをチェックする
- .exampleファイルの内容が実際のファイルと乖離しないかをチェックするスクリプト
    - 例えば`config.example.yaml`の内容が`config.yaml`と乖離していないかをチェックする

# 課題管理 INDEX

## 目的

このフォルダでは、プロジェクト全体で認識されている課題・改善点を体系的に管理します。
課題を可視化し、優先度とステータスを明確にすることで、計画的な改善を促進します。

---

## 優先度の定義

| 優先度 | 説明 |
|--------|------|
| **高** | 開発効率や品質に直接影響。早期対応が望ましい |
| **中** | 改善すると効果が大きいが、急ぎではない |
| **低** | 時間がある時に対応。やる気が起きたタイミングで |

---

## ステータスの定義

| ステータス | 説明 |
|------------|------|
| 未着手 | まだ作業を開始していない |
| 進行中 | 現在対応中 |
| 完了 | 対応完了 |

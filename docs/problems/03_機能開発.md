# 機能開発に関する課題

機械学習、LLM など機能面の課題を管理します。

---

## FNC-001: 画像解析とデータ分析の統一インターフェース

| 項目 | 内容 |
|------|------|
| **ID** | FNC-001 |
| **優先度** | 中 |
| **ステータス** | 未着手 |

### 現状
画像解析と数値データ分析が完全に独立している。
ドメインが大きく異なるため、ある程度は仕方がない状況。

### 改善案
- 利用する側に統一されたインターフェースを提供
- 共通の入出力形式を定義
- ファサードパターンの導入を検討

#NOTE: 抽象化レイヤーの設計が必要
理由: 呼び出し側が個別の実装詳細を意識しなくて済むようにすることで、保守性と拡張性が向上する

## FNC-002: 画像解析の学習・評価フローの改善

| 項目 | 内容 |
|------|------|
| **ID** | FNC-002 |
| **優先度** | 中 |
| **ステータス** | 未着手 |

### 現状
画像解析（IMAGE フォルダ）の学習・評価の流れが分かりにくい。
コード分割も不十分。

### 改善案
- 学習・評価フローのドキュメント整備
- コードの適切な分割（データ読み込み、前処理、学習、評価）
- パイプラインの可視化

#NOTE: 処理フローを明確にドキュメント化すべき
理由: 新規参入者が学習・評価の流れを理解するのに時間がかかる現状がある

## FNC-003: クラウド分散学習・並列推論

| 項目 | 内容 |
|------|------|
| **ID** | FNC-003 |
| **優先度** | 低 |
| **ステータス** | 未着手 |

### 現状
クラウド上での分散学習・並列推論の仕組みがない。

### 改善案
- AWS/Azure/GCP での分散学習環境の構築
- 並列推論用のインフラ設計
- スケーラビリティの確保

#NOTE: まずはローカルでの並列処理から始めるべき
理由: クラウド環境構築の前に、ローカルでの並列処理パターンを確立しておくと移行がスムーズ

## FNC-004: 他種類の機械学習の導入

| 項目 | 内容 |
|------|------|
| **ID** | FNC-004 |
| **優先度** | 低 |
| **ステータス** | 未着手 |

### 現状
画像解析と数値データ分析以外の機械学習が未対応。

### 改善案
以下の種類の機械学習を導入検討：
- Sentiment Analysis（感情分析）
- 時系列予測
- 強化学習

#NOTE: ユースケースを先に明確化すべき
理由: 技術先行で導入すると、活用されないコードが増えるリスクがある

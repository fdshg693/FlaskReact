# 開発ツールに関する課題

AI活用、カスタムスクリプト、開発効率化に関する課題を管理します。

---

## DEV-001: GitHub Copilot Chat のコンテキスト肥大化

| 項目 | 内容 |
|------|------|
| **ID** | DEV-001 |
| **優先度** | 中 |
| **ステータス** | 未着手 |

### 現状
GitHub Copilot Chat では、ワークスペース全体のファイル構成や使用可能なツール情報が自動的にAIに渡される。
- ワークスペースが大規模になると、コンテキストが肥大化
- AIの応答品質が低下する可能性がある
- カスタムエージェントである程度対応可能だが、ファイル数の肥大化には対策が難しい

### 改善案
- カスタムエージェントの活用
- 必要に応じてワークスペースの分割を検討
- `.github/copilot-instructions.md` の最適化

#NOTE: ワークスペース構成の見直しを検討すべき
理由: プロジェクトが成長するにつれ、AI活用の効率が低下するリスクがある

## DEV-002: CLI ベース AI エージェントの活用

| 項目 | 内容 |
|------|------|
| **ID** | DEV-002 |
| **優先度** | 中 |
| **ステータス** | 未着手 |

### 現状
CLI でのやりとりではワークスペース全体の情報が渡らず、コンテキストは抑えられる。
ただし、まだ発展途上であり、細かいカスタマイズはスクリプトなどで対応が必要。

### 改善案
- CODEX などの CLI ベース AI エージェントを活用
- コードレビューやドキュメント生成の自動化
- コミット単位・PR 単位での AI 自動化

### GitHub Copilot のさらなる活用
- コメント追加など細かい作業用エージェント作成
- GIT のコミットごとにレビューできるエージェント・プロンプト
- フォルダ構成を跨いだ問題を処理するためのエージェント・プロンプト・メモリ用ファイル保存場所

#NOTE: CLI ツールのラッパースクリプトを整備すべき
理由: 現状のCLIツールは設定が煩雑で、チーム内での共有が難しい

## DEV-003: MCP サーバーの活用

| 項目 | 内容 |
|------|------|
| **ID** | DEV-003 |
| **優先度** | 低 |
| **ステータス** | 未着手 |

### 現状
MCP サーバーを用いた専用ツールの AI 活用がまだ行われていない。

## DEV-004: クラウド AI サービスの活用

| 項目 | 内容 |
|------|------|
| **ID** | DEV-004 |
| **優先度** | 低 |
| **ステータス** | 未着手 |

### 現状
Azure・AWS などのクラウド AI サービスを活用していない。

### 改善案
- Lambda 関数などを用いてレビューやドキュメント生成用コードをマイクロサービス化

#NOTE: コスト試算と効果測定の計画が必要
理由: クラウドサービスは従量課金のため、導入前にROIを見積もる必要がある

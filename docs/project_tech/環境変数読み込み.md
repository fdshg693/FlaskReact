# 環境変数の読み込み方

## 概要
基本的に、以下の優先順位で環境変数を設定する

1. システム環境変数
2. `.env`ファイルの内容
    - `load_dotenv_workspace`関数を使用して読み込む
    - 基本的にはoverride=False（デフォルト）のまま使い、上書きはしない

## 使い方
```python
from config.load_setting import load_dotenv_workspace
load_dotenv_workspace()  # 環境変数を読み込む、環境変数の上書きはしない
``` 

## 読み込み場所
原則、サーバー起動時に一度だけ読み込めば十分なので、起動ファイルまたはアプリ初期化部分で実行する。

### その他、読み込みが必要な箇所
- ```if __name__ == "__main__":``` ブロック内でスクリプトを直接実行する場合  
    - この場合は、インポート・読み込みを全て__main__ブロック内に入れることを推奨する(他のモジュールからインポートされたときに実行されないようにするため)
- `exmaples/`, `experiments/`などのサンプルコード・実験コード内  
    - これらのコードは、プロジェクトの他の部分からインポートされることを想定していないため、各スクリプト内で明示的に読み込みを行って問題ない

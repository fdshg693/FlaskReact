# 開発規約 INDEX

## 目的

本開発規約は、プロジェクトの品質と開発効率を維持・向上させるために、チームメンバーが共通して従うべきルールとガイドラインをまとめたものです。

### 規約の原則

- **実用性重視**: 過度に厳格なルールは設けず、開発効率を阻害しない範囲で品質を担保する
- **学習と開発の両立**: 本プロジェクトは学習目的も含むため、実験的なコードと本番コードを明確に分離する
- **継続的改善**: 規約自体も定期的に見直し、改善していく

## 使い方

### 新規参加者向け

1. まずこのINDEX.mdを読み、規約の全体像を把握する
2. [01_プロジェクト構造.md](./01_プロジェクト構造.md) でフォルダ構成を理解する
3. 開発を始める前に [06_Git運用.md](./06_Git運用.md) でブランチ戦略を確認する
4. コードを書く際は [02_コーディング規約.md](./02_コーディング規約.md) を参照する

### 日常の開発時

- 新しいデータを追加する → [03_データ管理.md](./03_データ管理.md)
- テストを追加する → [04_テスト戦略.md](./04_テスト戦略.md)
- ドキュメントを作成する → [05_ドキュメント管理.md](./05_ドキュメント管理.md)

### 規約の更新時

規約の更新は、PRを通じてチームでレビューしてから反映する。

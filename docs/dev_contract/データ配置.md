# データ配置に関するガイドライン

## 元データ
- 元データを```data```に配置する
- どこのサービスから使われることを想定しているか、わかりやすいフォルダ構成にすること。

## ２次データ（再利用される可能性があるデータ）
- これらのデータから作成される２次データは、原則として```outputs```フォルダに配置する。
- ```outputs```は原則として、GIT管理下に置かないが、出力ファイルが分かりやすく分類されるように、パスを工夫すること。

## 一時データ（再利用されないデータ）
- 再利用されることを想定しない、一時的な出力データは、```tmp```フォルダに配置する。
- ```tmp```フォルダはGIT管理下に置かれていない。

# フォルダ構造

原則として、以下のようなフォルダ命名規則・構造に従うことが推奨される。

- experiments/ : 各種実験用スクリプトを配置するフォルダ
- examples/ : アプリ本体でも活用する関数・クラスなどの使い方を示すサンプルスクリプトを配置するフォルダ
- models/ : アプリ本体で活用するモデル関連の関数・クラスなどを配置するフォルダ
- utils/ : アプリ本体で活用するユーティリティ関数・クラスなどを配置するフォルダ
- core/ : アプリ本体で活用する中核的な関数・クラスなどを配置するフォルダ
- entry/ : Webアプリケーションのエントリポイント・機械学習などの推論エントリポイントを配置するフォルダ

## 設定ファイルの配置

- パス
    - パス関係は、```src/config/paths.py```で一元管理
- 秘密情報
    - APIキーなどの機密情報はルート直下の`.env`に書いて、```src/config/load_setting.py```で読み込む
        - 当分は予定がないが、複雑化したら、各サービス・フォルダごとに`.env`を分けることも検討
-　機械学習関連
    - サービスごとに様々なので、それぞれのサービス・機能単位で`config`フォルダを作成し、その中に配置する
    - YAMLなどの設定ファイルはGIT管理から外して、読み込みに使うLOADERクラスだけGIT管理する

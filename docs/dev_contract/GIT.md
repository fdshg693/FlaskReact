# GIT

## ブランチ
原則、devブランチからそれぞれ、maikuma, sekishiroのようにブランチをきる（分かりやすさ重視で、自分の名前をそのままでいい）. 
maikumaがタイミングを見て、devからmainにマージする. 
切りのいいところで、Github上で、devブランチへのPRを投げる. 

## GIT管理に追加するもの・しないもの
- データ関連
    - 一次データは基本的に、他の人にも共有しやすいように、GIT管理に追加する.
        - ただし、容量が大きいものは、GIT管理に追加しない. 
        - 例えば、学習済みモデルのファイルなどは、GIT管理に追加しない. 
    - 一次データから作られる、二次データは、原則として、GIT管理に追加しない. 
        - ただし、他の人がすぐに使えるようにするために、二次データをGIT管理に追加することもある. 
        - 例えば、前処理済みのデータなどは、GIT管理に追加することがある.
- ドキュメント関連
    - 個人ドキュメント管理のためのフォルダ（sekishiro, maikuma, seiwan, tamashiro, taishi）は、GIT管理に追加しない. 
    - 共同ドキュメント管理のためのフォルダ（docs）は、GIT管理に追加する.

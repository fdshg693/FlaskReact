# ドキュメントの書き方

## 人間用ドキュメント
README.mdおよび、`docs`配下のフォルダ（`docs/rule_docs_for_llm`を除く）に配置されるドキュメントは、人間が読むことを想定したドキュメントとする。
- 基本的にMarkdown形式で記載する。
- 理想では、図や表を多用し、視覚的に理解しやすい
    - AIを活用して図や表を生成できると理想

## AI用ドキュメント

### Github Copilot用ドキュメント
- Github Copilotに常に自動で渡るルート直下の`.github/`配下のInstructionsの内容は、常に高精度を保つ。  
    - その為に、細かい情報は除外して、必要最低限のビッグピクチャーのみが入るようにする。      
    - あくまで、AIが読むようなので精度向上のため英語で記載する。
        - 上記を満たせるように、自動でAIがその通りに生成する仕組みを作成する。   

### LLM全般用ドキュメント
#### 人間作成
`docs/docs_for_llm/`配下のドキュメントは、ドキュメントの中でも特にLLMの知識に組み込みたい内容を記載する。  
プロジェクト概要や開発規約・ある機能の詳細などを、人間が読みやすい形でまとめる。

AIを用いて生成するのは構わないが、常に人間が管理・監修する。

#### AI自動生成
`docs/llm_memory`配下のドキュメントは、AIが自動で生成・更新するドキュメントとする。  
ここには、AIのコード調査の結果の要約、コード編集内容の要約、`docs/docs_for_llm/`配下のドキュメントのAI用カスタマイズ版などを配置する。
一時タスク用のドキュメント・プロジェクト全体の知識ベースなどが混ざらないように注意する。
これらのドキュメントを、Github Copilotの指示に適宜含めることで、AIの理解度を高めることを目的とする。

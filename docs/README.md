# docsフォルダの構成

- dev_contract/
    - 開発規約をまとめたフォルダ
- new_challenge/
    - これから挑戦したい技術やアイデアをまとめたフォルダ
- problems/
    - 課題や問題点をまとめたフォルダ
- project_overview/
    - 現在のプロジェクト固有のツールや設定をまとめたフォルダ
- docs/rule_docs_for_llm/
    - AIに開発規約などを理解させるためのドキュメントをまとめたフォルダ（現状、未稼動）
- docs/tech_knowledge/
    - 技術的な知識やノウハウをまとめたフォルダ
    - 汎用的な内容はGIT管理して、プロジェクト固有の内容はproject_overviewに分け、個人的な勉強用の内容は`your_name/`フォルダ（GIT管理されない）を作成してまとめる

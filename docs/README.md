# ドキュメント管理 - README

このフォルダは、プロジェクトに関するすべてのドキュメントを体系的に管理するための場所です。

---

## 📁 フォルダ構成

### 人間用ドキュメント（日本語）

| フォルダ | 目的 | 管理方針 | リンク |
|---------|------|---------|--------|
| **project_overview/** | プロジェクト全体像の理解 | プロジェクト固有情報。新規メンバー向け | [INDEX](./project_overview/00_INDEX.md) |
| **dev_contract/** | 開発規約・ルール | チーム共通のルール。Git管理 | [INDEX](./dev_contract/INDEX.md) |
| **problems/** | 課題・改善点の管理 | 優先度とステータスで体系的に管理 | [INDEX](./problems/INDEX.md) |
| **tech_knowledge/** | 技術的な知識・ノウハウ | 汎用的な技術情報。Git管理 | [フォルダ](./tech_knowledge/) |

### AI用ドキュメント（英語）

| パス | 目的 | 状態 |
|------|------|------|
| **`.ai/knowledge/`** | LLMのコンテキストメモリ | 未稼働（将来的にAIが自動管理） |

### 個人用ドキュメント（Git管理外）

個人的な学習メモや実験内容は、`docs/tech_knowledge/your_name/` フォルダを作成して管理してください。

```
docs/tech_knowledge/
  ├── general/        # Git管理：汎用的な技術知識
  ├── python/         # Git管理：Python関連
  ├── react/          # Git管理：React関連
  └── your_name/      # Git管理外：個人的な学習メモ (.gitignore に追加)
```

## 📝 ドキュメント作成ガイドライン

### 基本原則

1. **最小限の場所に書く**
   - 関連フォルダの直下に `README.md` や `IMPORTANT.md` で済むなら、それを優先
   - `docs/` フォルダは体系的な整理が必要な場合のみ使用

2. **言語の使い分け**
   - **人間用**: 日本語（チームメンバーの理解を最優先）
   - **AI用**: 英語（AIが自動生成・自動更新）

3. **AI活用**
   - 可能な限り、AIによる自動生成・自動更新の仕組みを整備
   - GitHub Copilot、Claude、その他LLMツールを積極活用

### ドキュメント品質の維持

- **定期的な見直し**: 最低でも月1回は古い情報をチェック
- **実装との整合性**: コード変更時は関連ドキュメントも更新
- **テンプレート活用**: 各フォルダのINDEX.mdをテンプレートとして参考に

## 📊 ドキュメント状態の管理

### ステータス表示

各ドキュメントには、以下のステータスを明記することを推奨：

| ステータス | 意味 | 表示例 |
|-----------|------|--------|
| 🟢 最新 | 情報が正確で最新 | `[最終更新: 2025-11-30]` |
| 🟡 要更新 | 一部古い情報がある | `⚠️ 要更新: XX機能が変更済` |
| 🔴 非推奨 | 使用しない情報 | `❌ 非推奨: 代わりにXXを参照` |

### ドキュメントの依存関係

- ドキュメント間で相互参照する場合は、相対リンクを使用
- 循環参照を避け、階層的な構造を維持
- 重複した情報は一箇所にまとめ、他はリンクで参照

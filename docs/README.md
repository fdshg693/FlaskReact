# docsフォルダの構成

## 人間用ドキュメント
- dev_contract/
    - 開発規約をまとめたフォルダ
- new_challenge/
    - これから挑戦したい技術やアイデアをまとめたフォルダ
- problems/
    - 課題や問題点をまとめたフォルダ
- project_overview/
    - 現在のプロジェクト固有のツールや設定をまとめたフォルダ
- tech_knowledge/
    - 技術的な知識やノウハウをまとめたフォルダ
    - 汎用的な内容はGIT管理して、プロジェクト固有の内容はproject_overviewに分け、個人的な勉強用の内容は`your_name/`フォルダ（GIT管理されない）を作成してまとめる

## AI用ドキュメント
- docs_for_llm/
    - AIに開発規約などを理解させるためのドキュメントをまとめたフォルダ（現状、未稼動）
        - 人間がメンテナンスすることを想定
        - LLMが直接参照するわけではなく、ここのドキュメント内容をもとに`.ai/knowledge`フォルダ内のドキュメントを生成・更新するイメージ
        - 日本語
- `.ai/knowledge`(別ディレクトリに存在)
    - LLMのメモリとして使用するドキュメントをまとめたフォルダ（現状、未稼動）
    - LLMが自動でメンテナンスすることを想定
    - 英語

## 基本的なドキュメントの書き方
- もし、少ないファイル数で済むならば、`docs`フォルダに書かず、関連フォルダの直下の`README.md`や`IMPORTANT.md`などに書くことを優先する
- 人間用のドキュメントは日本語で、AI用のドキュメントは英語でAI自身に生成させることを優先する
- 可能な限り、AIによる自動生成・自動更新ができる仕組みを整備する
